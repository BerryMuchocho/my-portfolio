# 💼 Frontend Developer Portfolio (React + TailwindCSS)

## 📌 Overview

This project is a personal frontend developer portfolio built using **React** and **TailwindCSS**.

It showcases selected projects, UI design decisions, and my approach to building clean, responsive, and user-focused interfaces.

The goal of this project was to apply component-based architecture, routing, and modern styling to a real-world product.

---

## 🎯 Objectives

* Build a responsive portfolio for real-world use (job applications)
* Practice component-based UI design in React
* Implement client-side routing using React Router
* Use TailwindCSS to create a consistent design system
* Improve problem-solving through hands-on development

---

## 🚀 Features

* Responsive layout (mobile-first approach)
* Project gallery with reusable card components
* Navigation using React Router
* Clean UI built with Tailwind utility classes
* Structured component hierarchy for scalability

---

## 🛠️ Tech Stack

* **React (Vite)** – UI development and component structure
* **TailwindCSS** – Utility-first styling
* **React Router** – Page navigation
* **JavaScript (ES6+)** – Application logic

---

## ⚙️ Installation & Setup

### 1. Clone the repository

```bash
git clone https://github.com/BerryMuchocho/my-portfolio.git
cd my-portfolio
```

### 2. Install dependencies

```bash
npm install
```

### 3. Run the development server

```bash
npm run dev
```

### 4. Open in browser

```bash
https://my-portfolio-alpha-sepia-72.vercel.app/
```

## ⚠️ Challenges & Solutions

### 1. Components rendering twice (React StrictMode)

* **Issue:** Components rendered twice in development
* **Cause:** StrictMode behavior in React
* **Solution:** Adjusted debugging approach and understood why it happens

### 2. Tailwind styles not applying

* **Issue:** Styling not reflected in UI
* **Cause:** Incorrect configuration / missing imports
* **Solution:** Fixed Tailwind setup and ensured correct CSS integration

### 3. Layout inconsistencies across screen sizes

* **Issue:** UI not consistent on different devices
* **Solution:** Refined spacing, sizing, and responsive classes using Tailwind
